See Orchestrator in Action

The Configurability Engine

Your screens. Your questions. Your rules. Your API calls. No IT ticket. No release cycle. Changes go live when you say so.

How it works

You configure. Enroller executes.

Orchestrator is the control layer behind Pylaex Enroller. Every screen your agents see, every question asked, every API call made — it all starts here, configured by your operations team, not an engineering sprint.

What you control

Screen order and flow. Which questions appear and in what order. API calls — which to make, when, and when to bypass. Underwriting logic: approve, decline, refer, reflexive triggers. Rate tables and eligibility rules.

What you don't need

A developer. An IT ticket. A release cycle. A testing sprint. Configure in sandbox, preview it in Enroller, promote to production in minutes. Your business team owns the product.

Configure everything without writing code

Screen Order

Drag and drop to reorder screens in the flow. Changes reflect in Enroller immediately after you promote from sandbox.

Questions & Branching

Activate or suppress questions within any screen. Set reflexive logic — yes to X automatically surfaces follow-up questions Y and Z.

API Calls

Toggle external data calls on or off per product or eligibility type. Control when to call, what to do with results, and when to bypass entirely.

Underwriting Logic

Configure approve, decline, refer, and table-rating outcomes based on responses and API results. Every decision is logged and auditable.

Product Rules

Rate tables, eligibility windows, benefit tiers — configurable without a release cycle. Update and promote in minutes, not quarters.

Sandbox & Promotion

All changes happen in sandbox first. Preview in Enroller, validate, then promote. Your team controls the go-live timeline.
